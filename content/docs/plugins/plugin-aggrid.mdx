Data grid component powered by AG Grid Community Edition with lazy loading for optimal performance.

## Installation

```bash
npm install @object-ui/plugin-aggrid ag-grid-community ag-grid-react
```

Note: `ag-grid-community` and `ag-grid-react` are peer dependencies and must be installed separately.

## Features

- **Lazy Loading**: AG Grid libraries are loaded on-demand for optimal performance
- **Zero Configuration**: Just import the package and use `type: 'aggrid'` in your schema
- **Automatic Registration**: Components auto-register with the ComponentRegistry
- **Full AG Grid Features**: Sorting, filtering, pagination, row selection, and more
- **Multiple Themes**: Quartz (default), Alpine, Balham, and Material
- **Customizable**: Full access to AG Grid's GridOptions for advanced configuration

## Schema API

```typescript
{
  type: 'aggrid',
  rowData?: any[],                              // Grid data (required)
  columnDefs?: ColDef[],                        // Column definitions (required)
  pagination?: boolean,                         // Enable pagination (default: false)
  paginationPageSize?: number,                  // Rows per page (default: 10)
  theme?: 'quartz' | 'alpine' | 'balham' | 'material',  // Grid theme (default: 'quartz')
  height?: number | string,                     // Grid height (default: 500)
  rowSelection?: 'single' | 'multiple',         // Row selection mode
  domLayout?: 'normal' | 'autoHeight' | 'print', // Layout mode
  animateRows?: boolean,                        // Animate row changes (default: true)
  gridOptions?: GridOptions,                    // Advanced AG Grid options
  className?: string                            // Tailwind classes
}
```

## Column Definition Properties

| Property | Type | Description |
|----------|------|-------------|
| `field` | string | The field in the data to display |
| `headerName` | string | The header label for the column |
| `sortable` | boolean | Enable sorting for this column |
| `filter` | boolean \| string | Enable filtering (`true`, `'text'`, `'number'`, `'date'`) |
| `width` | number | Fixed column width in pixels |
| `checkboxSelection` | boolean | Show checkbox for row selection |
| `pinned` | 'left' \| 'right' | Pin column to left or right side |
| `valueFormatter` | function | Custom formatter for cell values |
| `cellRenderer` | function | Custom renderer for cell content |

## Usage

### Basic Setup

```typescript
// In your app entry point (e.g., App.tsx or main.tsx)
import '@object-ui/plugin-aggrid';

// Now you can use aggrid type in your schemas
const schema = {
  type: 'aggrid',
  rowData: [
    { make: 'Tesla', model: 'Model Y', price: 64950 },
    { make: 'Ford', model: 'F-Series', price: 33850 },
    { make: 'Toyota', model: 'Corolla', price: 29600 }
  ],
  columnDefs: [
    { field: 'make', headerName: 'Make', sortable: true, filter: true },
    { field: 'model', headerName: 'Model', sortable: true, filter: true },
    { field: 'price', headerName: 'Price', sortable: true, filter: 'number' }
  ],
  pagination: true,
  paginationPageSize: 10,
  theme: 'quartz',
  height: 500
};
```

### TypeScript Support

```typescript
import type { AgGridSchema } from '@object-ui/plugin-aggrid';

const schema: AgGridSchema = {
  type: 'aggrid',
  rowData: [...],
  columnDefs: [...],
  pagination: true,
  theme: 'quartz'
};
```

### Advanced Configuration

```typescript
const schema = {
  type: 'aggrid',
  rowData: [...],
  columnDefs: [...],
  gridOptions: {
    suppressCellFocus: true,
    enableCellTextSelection: true,
    enableRangeSelection: true,
    rowMultiSelectWithClick: true,
    // Any AG Grid GridOptions property
  }
};
```

## Performance

The plugin uses lazy loading to minimize bundle size impact:

- **Initial load**: +0.19 KB (entry point only)
- **On render**: +200-300 KB (AG Grid library loaded on-demand)

This results in significantly faster initial page loads for applications that don't use data grids on every page.
