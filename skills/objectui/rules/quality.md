# Quality Rules

## Validation workflow

- Keep skills/docs changes traceable and indexable from the root `SKILL.md`.
- Run lint/build/tests according to repository workflow before finalizing skill migrations.
- Keep generated/release-related commands aligned with existing project scripts.

## Cross-domain linkage requirements

- Every guide must be reachable from root guide index.
- Cross-topic rules must link to at least one related guide for agent retrieval.
- Remove redundant per-skill frontmatter in split guide/rule files; keep only root skill frontmatter.
